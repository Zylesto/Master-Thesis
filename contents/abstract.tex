This master's thesis introduces a classification and recommendation concept for software vulnerabilities by integrating multiple severity dimensions. Leveraging \ac{CVSS} for technical impact and \ac{EPSS} for exploit likelihood, the framework yields a dynamic perspective on both intrinsic risk and real-world threats. We first propose a model and algorithm to compute classification scores for known vulnerabilities within specific software contexts. A second model defines role-specific remediation strategies informed by patch availability, asset criticality, stakeholder requirements, and additional contextual factors, utilizing the \ac{SSVC} framework to enable nuanced guidance. A third model establishes a rank-ordering of all classified vulnerabilities, highlighting critical or data-incomplete cases for immediate attention. The implementation draws from \ac{OSV} and other public sources, aided by caching and time-based invalidation to respect external rate limits. Evaluation shows that this integrated approach successfully addresses the wide-ranging challenges posed by extensive \ac{SBOM}s, offering a transparent, scalable, and context-aware solution for managing open-source security risks.
